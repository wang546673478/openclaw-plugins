# Session Memory Skill

双重机制确保重要上下文不丢失：
1. **定期保存**：每30分钟自动保存会话摘要
2. **查询结束提取**：连续5轮无tool call时，提取关键记忆

---

## 状态文件

读取 `memory/.session-memory-state.json` 获取上次保存状态。

---

## 机制一：定期保存

### 检查条件

距上次保存 **≥ 30 分钟**

### 执行步骤

1. 通过 `session_status` 获取当前会话状态
2. 生成会话摘要，格式：

```markdown
## HH:mm 会话摘要

- **任务**：用户做了什么/要做什么
- **决定**：有哪些关键决策（代码、设计、技术选型）
- **下一步**：接下来要做什么
```

3. 追加到 `memory/YYYY-MM-DD.md`
4. 更新 `memory/.session-memory-state.json`

---

## 机制二：查询结束提取（ExtractMemories）

### 触发条件

**连续 5 轮无 tool call**

这意味着用户可能完成了一个独立的任务或查询，适合提取关键信息。

### 执行步骤

1. 读取最近 10-15 条消息，识别"已完成的任务"
2. 提取关键信息：
   - 任务目标（用户想完成什么）
   - 解决方案（做了什么、用了什么方法）
   - 重要决策（技术选型、代码决策）
   - 结果（成功/失败/待处理）

3. 保存格式：

```markdown
## HH:mm 记忆提取

### 任务
<1-2句话描述>

### 解决方案
<关键做法>

### 决策
<重要决策点>

### 结果
<成功/失败/待处理>
```

4. 追加到 `memory/YYYY-MM-DD.md`

---

## 机制三：跨会话整合（Auto-Dream）

### 触发条件

- 距上次整合 **≥ 24 小时**
- 此期间有 **≥ 5 个新会话**
- 没有整合任务在运行

### 执行步骤

1. **检查锁**：读取 `memory/.dream-lock.json`，如果锁存在且未过期则跳过
2. **获取锁**：写入当前时间到 `memory/.dream-lock.json`
3. **收集材料**：`sessions_list` 获取活跃会话，读取各 `memory/YYYY-MM-DD.md`
4. **识别模式**：
   - 重复出现的主题
   - 未完成的任务
   - 技术决策记录
   - 长期目标
5. **生成整合报告**：

```markdown
## 梦境整合报告 - YYYY-MM-DD

### 主题聚合
<识别出的3-5个核心主题>

### 未完成任务
<上次提到的、待完成的事项>

### 决策记录
<技术决策及其时间线>

### 下一步
<基于模式的预测>
```

6. **更新状态**：写入 `memory/.dream-state.json`
7. **释放锁**：删除 `memory/.dream-lock.json`

---

## 注意事项

- 摘要要**简洁**，1-3段话
- 关注**决策和下一步**，不要复述过程
- 多次机制可以同时触发，都是**追加**模式
- 不要在摘要中提及"这次对话是 AI 生成的摘要"这类废话
