# Coordinator Skill

当用户要求同时处理多个独立任务时，作为协调者工作。

## 使用场景

- 用户说"帮我做 A 和 B 和 C"
- 用户说"这几个事情可以一起处理"
- 一个大任务可以并行拆解

## 工作流程

### 1. 分解任务

分析用户需求，拆解为**独立子任务**：

- 每个子任务应该能独立运行
- 子任务之间无依赖或依赖最小化
- 识别共享上下文（哪些子任务需要相同的信息）

### 2. 准备上下文

确定每个子任务需要的：
- 任务描述（精确的 instruction）
- 共享上下文（如代码库状态、文件内容）
- 输出格式要求

### 3. 并行执行

使用 `sessions_spawn` 启动多个 subagent：

```typescript
// 伪代码示例
const results = await Promise.all([
  sessions_spawn({ task: "子任务1描述", runtime: "subagent", mode: "run" }),
  sessions_spawn({ task: "子任务2描述", runtime: "subagent", mode: "run" }),
  sessions_spawn({ task: "子任务3描述", runtime: "subagent", mode: "run" }),
]);
```

### 4. 汇总结果

等待所有子任务完成，汇总为统一回复：

```markdown
## 任务汇总

### ✅ 子任务1
<简短结果>

### ✅ 子任务2
<简短结果>

### ✅ 子任务3
<简短结果>

---

共 <N> 个任务，失败 <M> 个
```

### 5. 错误处理

- 如果某个子任务失败，继续其他任务
- 最终汇总时标注失败的任务及原因
- 如果全部失败，明确告知用户

## 限制

- 并行数量建议 ≤ 5 个（太多会混乱）
- 每个子任务应该简单明确
- 不适合有强依赖顺序的任务

## 示例 Prompt

```
用户需求：帮我分析这个项目的 API 结构，同时生成 API 文档，然后写测试

分解：
1. 分析 API 结构（read 文件，找 endpoints）
2. 生成文档（基于分析结果）
3. 写测试（基于 API 结构）

并行执行 1，
等 1 完成后并行执行 2 和 3
```
